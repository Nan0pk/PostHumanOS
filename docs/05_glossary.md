# Glossary

## Admissibility

The property that an action is allowed to execute because it passed deterministic checks, proofs, sandbox simulation, policy validation, or another explicit admission gate.

## Capability Contract

A machine-readable description of a component's functions, limits, telemetry, authority requirements, and operational constraints.

## Deterministic Admission Gate

A non-probabilistic mechanism that checks whether a proposed action can alter machine state.

## Econometric Orchestration

Planning and scheduling based on multiple resource currencies rather than only CPU time or priority.

## Formal Intent

A typed representation of a desired outcome with constraints, budgets, fulfillment tests, and rollback semantics.

## Governance Replacement

The requirement that any removed abstraction must have its accountability, audit, provenance, compatibility, and recovery functions replaced.

## Multi-Currency Resource Budget

A budget involving resources such as time, joules, thermal headroom, memory, accelerator occupancy, storage durability, network quality, and AI/model quota.

## Post-Syntactic Execution

Execution workflows where human-readable source is not the only central artifact. In PostHumanOS this is admissible only through traceability, verification, and provenance.

## Reflex Tier

A local, low-latency model or heuristic layer used for bounded classification, dispatch, and routine orchestration support.

## Reasoning Tier

A higher-capacity model or analysis layer used for complex planning, synthesis, or explanation, still subordinate to deterministic admission.

## Semantic State Graph

A graph representation of persistent state, relationships, provenance, rollback points, and compatibility projections.

## Transition Economics

Explicit accounting for the cost and benefit of reconfiguration, migration, synchronization, validation, and teardown.

## Universal Capability Contract / UCC

A proposed normalized contract format for describing hardware, software, service, model, or environmental capabilities in a scheduler- and verifier-readable form.
