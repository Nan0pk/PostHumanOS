# Core Concepts

## Formal Intent

A machine-checkable representation of a requested outcome, including goals, constraints, preferences, forbidden transitions, budgets, fulfillment tests, and rollback semantics.

## Universal Capability Contract

A declarative description of what a device, service, model, or subsystem can do, what limits it has, what telemetry it exposes, and what authority is required to use it.

## Multi-Currency Orchestration

Scheduling and planning across more than time or CPU cycles. Relevant currencies include joules, watts, thermal headroom, memory bandwidth, accelerator slots, network path quality, storage durability, and model quota.

## Deterministic Admission

A gate that decides whether a proposed state-changing action may run. Admission may be based on type checks, policy checks, sandbox simulation, proof-carrying code, model checking, translation validation, or restricted DSL verification.

## Semantic State Graph

A persistence model that stores state, relationships, provenance, lineage, compatibility projections, and rollback points as a graph rather than treating files as the only primary abstraction.

## Transition Economics

The accounting discipline for dynamic morphology. Reconfiguration has costs: formation, migration, synchronization, cache warmup, validation, proof checking, rollback, and teardown.
